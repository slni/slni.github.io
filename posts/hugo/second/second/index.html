一、根目录config.toml配置文件

通过下面的配置，完成对网站主体框架的配置搭建工作。

1. 基本配置

# 网站标题
title = "我的"
# 网站部署的网址
baseURL = 'https://slni.github.io/'
# 设置默认的语言 ["en", "zh-cn", "fr", "pl", ...]
defaultContentLanguage = "zh-cn"
# 网站语言, 仅在这里 CN 大写 ["en", "zh-CN", "fr", "pl", ...]
languageCode = "zh-CN"
# 是否包括中日韩文字
hasCJKLanguage = true
# 更改使用 Hugo 构建网站时使用的默认主题
theme = "FixIt"

[params]
  # FixIt 主题版本
  version = "0.2.X"

2. 导航菜单配置

  • 文章 - posts
  • 分类 - categories
  • 标签 - tags
[menu]
  [[menu.main]]
    identifier = "posts"
    name = "文章"
    url = "/posts/"
    weight = 1
  [[menu.main]]
    identifier = "categories"
    name = "分类"
    url = "/categories/"
    title = ""
    weight = 2
  [[menu.main]]
    identifier = "tags"
    name = "标签"
    url = "/tags/"
    weight = 3

3. 网站头部导航配置

[params.header]
# FixIt 0.2.13 | 更改 桌面端导航栏模式 ["sticky", "normal", "auto"]
desktopMode = "sticky"
# FixIt 0.2.13 | 更改 移动端导航栏模式 ["sticky", "normal", "auto"]
mobileMode = "auto"
  # FixIt 0.2.0 | 新增 页面头部导航栏标题配置
  [params.header.title]
  # LOGO 的 URL
  logo = "/images/hugo_logo.jpeg"
  # 标题名称
  name = "我的 Hugo FixIt 网站"

4. 网站页面底部信息配置

# 页面底部信息配置
[params.footer]
enable = true
# FixIt 0.2.17 | 更改 自定义内容（支持 HTML 格式）
# 进阶使用，见参数 `params.customFilePath.footer`
custom = ""
# FixIt 0.2.0 | 新增 是否显示 Hugo 和主题信息
hugo = true
# FixIt 0.2.0 | 新增 是否显示版权信息
copyright = true
# FixIt 0.2.0 | 新增 是否显示作者
author = true
# 网站创立年份
since = 2015
# FixIt 0.2.14 | 新增 是否显示网站内容总字数
wordCount = true
# FixIt 0.2.12 | 新增 公网安备信息，仅在中国使用（支持 HTML 格式）
gov = ""
# FixIt 0.2.12 | 更改 ICP 备案信息，仅在中国使用（支持 HTML 格式）
icp = ""
# 许可协议信息（支持 HTML 格式）
license = '<a rel="license external nofollow noopener noreferrer" href="https://creativecommons.org/licenses/by-nc/4.0/" target="_blank">CC BY-NC 4.0</a>'

5. Home页面配置

# 主页配置
[params.home]
# FixIt 0.2.0 | 新增 RSS 文章数目
rss = 10
  # 主页个人信息
  [params.home.profile]
    enable = true
    # Gravatar 邮箱，用于优先在主页显示的头像
    gravatarEmail = ""
    # 主页显示头像的 URL
    avatarURL = "/images/hugo_logo.jpeg"
    # FixIt 0.2.17 | 新增 头像菜单链接的 identifier
    avatarMenu = ""
    # FixIt 0.2.7 | 更改 主页显示的网站标题（支持 HTML 格式）
    title = "我的的个人网站"
    # 主页显示的网站副标题
    subtitle = "这是我的 Hugo FixIt 网站"
    # 是否为副标题显示打字机动画
    typeit = true
    # 是否显示社交账号
    social = true
    # FixIt 0.2.0 | 新增 免责声明（支持 HTML 格式）
    disclaimer = ""
  # 主页文章列表
  [params.home.posts]
    enable = true
    # 主页每页显示文章数量
    paginate = 6

6. 社交信息配置

[params.social]
    GitHub = "slni"
    // 更多完整的信息在themes/FixIt/config.toml中可以看到
    // ... 
    RSS = true

7. 解析文档的配置

[markup]
  # 语法高亮设置 (https://gohugo.io/content-management/syntax-highlighting)
  [markup.highlight]

    noClasses = false

二、文章内容的管理

1. 所有公共资源文件，存放到/static/images/目录中

2. 所有文章资源，存放到/content/posts/目录中

如上图，文章1的文件夹中存放着其对应的md文件和图片资源。

3. 如何引用图片资源文件

  1. 引用绝对图片路径(static目录中)： ![图片](/images/xxx.png)
  2. 引用相对图片路径(/content/posts/中，和md处于同一文件夹)： ![图片](../xxx.png)

三、文章内容前置参数

  1. 前置参数说明 以下是一个前置参数例子：
---
title: "我的第一篇文章"
date: 2020-03-04T15:58:26+08:00
draft: true
tags: []   # 文章的tag, 多个tag使用`,`进行分割
categories: [] # 文章的类别, 多个类别使用`,`进行分割
---
hello, 这是我第一篇文件的内容部分，😄。
  1. 前置参数，新增导航类别：
---
menu:
  main:
    parent: 导航类别A
    params:
      icon: fa-brands fa-readme
---

如上图图所示：

  • 文章：所有/content/posts/下的文章都在这里，包含所有
  • 分类：前置参数tags的文章
  • 标签: 前置参数categories的文章
  • 导航类别A：前置参数menu:main:parent:导航类别A的所有文章
